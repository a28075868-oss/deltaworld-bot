## ✅ ШАГ 1: Создайте таблицу donate_queue

Подключитесь к Railway MySQL и выполните этот SQL:

```bash
mysql -h reseau.proxy.rlwy.net -u root -pjlCtItyZXPAzzoLBwGlVDfxFWmkUXEtX --port 44404 railway
```

```sql
CREATE TABLE IF NOT EXISTS donate_queue (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nickname VARCHAR(255) NOT NULL,
  email VARCHAR(255),
  commands TEXT NOT NULL,
  price DECIMAL(10,2),
  status ENUM('pending','processing','completed','failed') DEFAULT 'pending',
  attempts INT DEFAULT 0,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  executed_at TIMESTAMP NULL,
  error_message TEXT,
  INDEX idx_status (status),
  INDEX idx_created (created_at),
  INDEX idx_nickname (nickname)
);
```

Или используйте HeidiSQL / phpMyAdmin с файлом `../DeltaWorld-NodeJS/config/add-donate-queue.sql`

## ✅ ШАГ 3: Деплой на Railway

1. Зайдите на https://railway.app/dashboard
2. **New Project** → **Deploy from GitHub repo** → Выберите `deltaworld-bot`
3. Добавьте **Environment Variables** (Settings → Variables):

```env
DB_HOST=reseau.proxy.rlwy.net
DB_PORT=44404
DB_USER=root
DB_PASSWORD=jlCtItyZXPAzzoLBwGlVDfxFWmkUXEtX
DB_NAME=railway

RCON_HOST=d20.aurorix.net
RCON_PORT=25575
RCON_PASSWORD=2557525575

CHECK_INTERVAL=10
MAX_ATTEMPTS=3
```

4. Railway автоматически задеплоит бота!

## ⚠️ ВАЖНО: RCON порт

Если видите ошибку **ECONNREFUSED**:

1. Порт 25575 закрыт на хостинге сервера
2. Попросите хостинг открыть этот порт
3. Проверьте `server.properties`: `enable-rcon=true`
4. Перезапустите сервер Minecraft

Если порт открыть невозможно → используйте плагин со стороны сервера (см. DEPLOY_GUIDE_RU.md, раздел "Альтернатива")
